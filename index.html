Notas Metodológicas

Cuándo elegir cada una

Scrum
Iteraciones fijas (sprints). Mejor para proyectos con alcance definido y necesidad de entregas periódicas predecibles.
Kanban
Flujo continuo sin sprints. Mejor para proyectos con requerimientos cambiantes; duración mayor pero más flexible.
Scrumban
Híbrido: Build con disciplina de sprints + Run con flujo Kanban continuo. Ideal para equipos que entregan Y mantienen.
XP
Enfoque en calidad: programación en pareja, pruebas automáticas antes del código e integración continua del trabajo. Mejor cuando un error en producción cuesta caro.
Water-Scrum-Fall
Híbrido tradicional+ágil: planificación formal inicial + desarrollo iterativo Scrum + despliegue formal final. Precio fijo cerrado. Ideal para empresas con procesos formales y gobierno corporativo.

Prácticas

Las 5 prácticas principales

P²

Programación en pareja

Dos desarrolladores en un teclado. Revisión continua, menos errores y transferencia de conocimiento del Senior al Junior en tiempo real.

✓

Pruebas antes del código

Primero se escribe el caso de prueba, después el código que lo cumple. Calidad asegurada desde el primer día y comportamiento verificable.

↺

Integración constante

Cada cambio en el código se valida automáticamente en minutos. Los errores se descubren temprano, no en semanas. Despliegues sin sorpresas.

↻

Limpieza continua del código

Mejora constante de la estructura interna sin cambiar lo que el sistema hace. La deuda técnica se paga sobre la marcha, no al final.

S

Diseño simple

La solución más simple que funcione. No construir lo que no se necesita hoy. Menos código = menos errores y más fácil de mantener.

⊕

Entregas pequeñas y frecuentes

Ciclos cortos de 2 semanas. Cada entrega aporta valor verificable al cliente, no solo avances técnicos internos.

K

Mayor rentabilidad: Kanban

XP

Recuperación más rápida: XP

4.85 meses para recuperar la inversión. Si liberar el dinero invertido pronto es la prioridad, XP es la mejor opción.

SB

Mejor balance: Scrumban

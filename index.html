Notas Metodológicas

Cuándo elegir cada una

Scrum
Iteraciones fijas (sprints). Mejor para proyectos con alcance definido y necesidad de entregas periódicas predecibles.
Kanban
Flujo continuo sin sprints. Mejor para proyectos con requerimientos cambiantes; duración mayor pero más flexible.
Scrumban
Híbrido: Build con disciplina de sprints + Run con flujo Kanban continuo. Ideal para equipos que entregan Y mantienen.
XP
Enfoque en calidad: pair programming, TDD, integración continua. Mejor para proyectos donde la calidad del código es crítica.
Water-Scrum-Fall
Híbrido tradicional+ágil: planificación cascada inicial + desarrollo Scrum + despliegue cascada final. Precio fijo cerrado. Ideal para empresas con procesos formales y gobierno corporativo.

Prácticas

Las 5 prácticas core

P²

Pair Programming

Dos desarrolladores en un teclado. Revisión continua, menos bugs, transferencia de conocimiento Senior → Junior en tiempo real.

TDD

Test-Driven Development

Primero el test, después el código. Cobertura alta desde el principio y diseño guiado por requerimientos verificables.

CI

Integración Continua

Cada commit dispara build + tests. Errores se detectan en minutos, no en semanas. Despliegues sin sorpresas.

↻

Refactoring

Mejora constante del código sin cambiar el comportamiento. La deuda técnica se paga continuamente, no al final.

S

Simple Design

La solución más simple que funcione. YAGNI: no construir lo que no se necesita ahora. Menos código = menos bugs.

⊕

Pequeños Releases

Sprints de 2 semanas estilo XP. Cada iteración entrega valor verificable al cliente, no solo features técnicas.

¿Cuál priorizar según PMI?

K

Mayor ROI absoluto: Kanban

139.3 % de retorno sobre la inversión. Si maximizar utilidad neta es la meta, Kanban genera Q 1,397,275 de beneficio.

XP

Mejor Payback: XP

4.85 meses para recuperar la inversión. Si la tesorería es la prioridad, XP libera capital antes que todas las demás.

SB

Mejor balance: Scrumban

ROI 115.5 %, Payback 6.03 m, costo más bajo (Q 423K). Si buscas eficiencia en todos los frentes, Scrumban es la elección PMI más equilibrada.

W

Cuando el contrato manda: WSF

ROI más bajo (67.9 %) pero único modelo de precio fijo cerrado. Necesario cuando el gobierno corporativo lo exige.

Volver al análisis comparativo

Las cifras PMI confirman: las cinco metodologías son financieramente viables. La elección depende de las prioridades del cliente.
